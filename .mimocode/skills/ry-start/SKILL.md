# ry-start

Use this skill when the owner asks MiMoCode to implement a non-trivial task.

Workflow:

1. Confirm the requested behavior and affected native surfaces.
2. Read code/config/tests before docs or memory.
3. Make a concise plan with verification commands.
4. Implement scoped changes.
5. Run targeted checks first, then release gates matching the touched scope.
6. Sync Serena memories and `MEMORY.md` only when verified current facts changed.

MiMoCode compose can structure the session, but it is not rldyour cmux
orchestrator mode.
